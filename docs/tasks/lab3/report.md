# Отчёт по лабораторной работе №3

## Текст задания

Вариант №36958: `Tutu.Ru` (`https://www.tutu.ru/`).

Необходимо:

- сформировать варианты использования сайта;
- разработать на их основе тестовое покрытие;
- провести функциональное тестирование интерфейса сайта;
- автоматизировать тестирование средствами Selenium;
- использовать XPath-селекторы для работы с динамически формируемыми элементами.

Примечание по реализации: Selenium RC является устаревшей технологией и в современной среде практически не используется. В проекте лабораторная работа реализована на актуальном стеке `Selenium WebDriver 4`, а шаблон Selenium IDE приложен в формате `.side`.

## UseCase-диаграмма

Файл диаграммы: `docs/tasks/lab3/use-cases.puml`.

Рассмотрены следующие прецеденты использования:

- просмотр основных сервисов путешествий на главной странице;
- переход в справочный раздел;
- просмотр блока `Идеи для поездок`;
- ввод направления в форме поиска отелей;
- использование блока подписки на рассылку.

## Описание набора тестовых сценариев

### TC-01. Отображение основных разделов на главной странице

Предусловие: браузер запущен.

Шаги:

1. Открыть сохранённую главную страницу `Tutu.ru`.
2. Дождаться загрузки главной страницы.
3. Проверить наличие блока `Идеи для поездок`.
4. Проверить наличие сервисов `Авиабилеты`, `Ж/д билеты`, `Автобусы`, `Отели`, `Электрички`.
5. Проверить корректность целевых ссылок сервисов.

Ожидаемый результат:

- главная страница отображается корректно;
- блок `Идеи для поездок` видим;
- основные сервисы присутствуют в DOM и ведут на корректные разделы сайта.

### TC-02. Переход в справочный раздел

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Найти ссылку `Справочная` в шапке сайта.
2. Проверить текст ссылки.
3. Проверить значение `href`.
4. Убедиться, что `href` содержит `/2read/`.

Ожидаемый результат:

- ссылка на справочный раздел доступна;
- URL ссылки содержит `/2read/`.

### TC-03. Ввод направления в форме поиска отелей

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Найти поле поиска отелей на сохранённой главной странице.
2. Ввести значение `Казань`.
3. Проверить, что значение сохранилось в поле.

Ожидаемый результат:

- поле принимает введённое значение;
- пользователь может задать направление поиска.

### TC-04. Проверка элементов блока подписки

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Прокрутить страницу к блоку подписки.
2. Ввести в поле email значение `invalid-email`.
3. Установить чекбокс согласия на обработку персональных данных.
4. Проверить доступность кнопки `Подписаться`.
5. Проверить, что введённое значение сохранено в поле.

Ожидаемый результат:

- поле email доступно пользователю;
- чекбокс согласия доступен и переключается;
- кнопка `Подписаться` присутствует;
- введённое значение сохраняется в поле.


Базовый URL по умолчанию:

- `file:///Users/macbook/ITMO/TPO/docs/tasks/lab3/site/main.html`

Причина выбора snapshot-подхода:

- живой `tutu.ru` нестабилен в headless-режиме из-за внешних ресурсов, гидрации и антиботных механизмов;
- сохранённый HTML-снимок фиксирует реальный DOM главной страницы и позволяет воспроизводимо запускать Selenium-проверки в `Chrome` и `Firefox`;
- XPath-локаторы строятся по содержимому и атрибутам элементов, а не по их `id`.

Текущий статус в рамках данной среды:

- тестовые сценарии подготовлены и структурированы через `Page Object`;
- XPath-локаторы используются во всех UI-проверках;
- сборка `./gradlew testClasses` успешно проходит;
- `./gradlew uiChromeTest -Dheadless=true` проходит успешно;
- `./gradlew uiFirefoxTest -Dheadless=true` проходит успешно.

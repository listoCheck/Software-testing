# Отчёт по лабораторной работе №3

## Текст задания

Вариант №36958: `Tutu.Ru` (`https://www.tutu.ru/`).

Необходимо:

- сформировать варианты использования сайта;
- разработать на их основе тестовое покрытие;
- провести функциональное тестирование интерфейса сайта;
- автоматизировать тестирование средствами Selenium;
- использовать XPath-селекторы для работы с динамически формируемыми элементами.

Примечание по реализации: Selenium RC является устаревшей технологией и в современной среде практически не используется. В проекте лабораторная работа реализована на актуальном стеке `Selenium WebDriver 4`, а шаблон Selenium IDE приложен в формате `.side`.

## UseCase-диаграмма

Файл диаграммы: `docs/tasks/lab3/use-cases.puml`.

Рассмотрены следующие прецеденты использования:

- просмотр основных сервисов путешествий на главной странице;
- переход в справочный раздел;
- просмотр блока `Идеи для поездок`;
- ввод направления в форме поиска отелей;
- ввод email для подписки на рассылку.

## Описание набора тестовых сценариев

### TC-01. Отображение основных разделов на главной странице

Предусловие: браузер запущен.

Шаги:

1. Открыть `https://www.tutu.ru/`.
2. Дождаться загрузки главной страницы.
3. Проверить наличие блока `Идеи для поездок`.
4. Проверить наличие элементов `Авиабилеты`, `Ж/д билеты`, `Автобусы`, `Отели`, `Электрички`.

Ожидаемый результат:

- главная страница отображается корректно;
- блок `Идеи для поездок` видим;
- основные сервисы доступны пользователю.

### TC-02. Переход в справочный раздел

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Нажать ссылку `Справочная` в шапке сайта.
2. Дождаться открытия страницы помощи.
3. Проверить URL страницы.
4. Проверить наличие заголовка `Справочная` и категории `Поезда`.

Ожидаемый результат:

- открывается страница справки;
- URL содержит `/2read/`;
- на странице есть заголовок `Справочная` и тематические категории.

### TC-03. Ввод направления в форме поиска отелей

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Найти поле `Город, отель или направление`.
2. Ввести значение `Казань`.
3. Проверить, что значение сохранилось в поле.

Ожидаемый результат:

- поле принимает введённое значение;
- пользователь может задать направление поиска.

### TC-04. Валидация email в форме подписки

Предусловие: открыта главная страница `Tutu.ru`.

Шаги:

1. Прокрутить страницу к блоку подписки.
2. Ввести в поле email значение `invalid-email`.
3. Нажать `Подписаться`.
4. Проверить HTML5-валидацию поля.

Ожидаемый результат:

- поле помечается как невалидное;
- отправка формы с некорректным email не выполняется.

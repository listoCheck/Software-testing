# Отчёт по лабораторной работе №3

## Текст задания

Вариант №36958: `Tutu.Ru` (`https://www.tutu.ru/`).

Необходимо:

- сформировать варианты использования сайта;
- разработать на их основе тестовое покрытие;
- провести функциональное тестирование интерфейса сайта;
- автоматизировать тестирование средствами Selenium;
- использовать XPath-селекторы для работы с динамически формируемыми элементами.

Примечание по реализации: Selenium RC является устаревшей технологией и в современной среде практически не используется. В проекте лабораторная работа реализована на актуальном стеке `Selenium WebDriver 4`, а шаблон Selenium IDE приложен в формате `.side`.

## UseCase-диаграмма

Файл диаграммы: `docs/tasks/lab3/use-cases.puml`.

В итоговой версии работы выделены 8 пользовательских прецедентов:

- UC-01. Просмотреть сервисный каталог и тематические ссылки в шапке.
- UC-02. Перейти в справочный раздел.
- UC-03. Изучить карточки блока `Идеи для поездок`.
- UC-04. Изучить FAQ-блок и перейти к полной базе знаний.
- UC-05. Просмотреть преимущества сервиса `Фишки Туту`.
- UC-06. Ввести направление в форме поиска отелей.
- UC-07. Заполнить форму email-подписки и подтвердить согласие.
- UC-08. Перейти к политике обработки персональных данных из формы подписки.

## Детализированные прецеденты

### Прецедент: HeaderNavigationAndCatalog

**ID:** UC-01

**Краткое описание:** гость сайта изучает основные сервисы путешествий и тематические ссылки в шапке главной страницы.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Навигационная подсистема сайта.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- DOM страницы успешно загружен.

**Основной поток:**

1. Гость открывает главную страницу сервиса.
2. Система отображает в шапке ссылки `Это выгодно!`, `Выходные`, `Маршруты`, `Афиша`, `Справочная`, `Путеводитель`.
3. Гость просматривает сервисный каталог.
4. Система отображает карточки сервисов `Авиабилеты`, `Ж/д билеты`, `Автобусы`, `Отели`, `Электрички`, `Туры`, `Аренда авто`, `Аэроэкспрессы`.
5. Гость убеждается в наличии кнопки `Войти`.

**Альтернативы / исключения:**

- 2a. Одна из ссылок в шапке отсутствует.
  Система не позволяет пользователю перейти к соответствующему разделу.
- 4a. Один из сервисов отсутствует или ведёт на некорректный URL.
  Пользователь не может начать сценарий выбора соответствующего вида путешествия.

### Прецедент: OpenHelpSection

**ID:** UC-02

**Краткое описание:** гость сайта переходит к справочному разделу из основной навигации.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Справочный раздел `2read`.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- ссылка `Справочная` присутствует в шапке или в блоке FAQ.

**Основной поток:**

1. Гость находит ссылку `Справочная` в шапке.
2. Система показывает целевой URL справочного раздела.
3. Гость использует ссылку для перехода к базе знаний.

**Альтернативы / исключения:**

- 1a. Ссылка `Справочная` недоступна.
  Гость не может перейти к информационному разделу.
- 2a. URL не содержит `/2read/`.
  Навигация уводит пользователя не в целевой раздел.

### Прецедент: ExploreTravelIdeas

**ID:** UC-03

**Краткое описание:** гость сайта изучает подборку направлений в блоке `Идеи для поездок`.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Контентный блок `Путеводитель`.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- отображается блок `Идеи для поездок`.

**Основной поток:**

1. Гость прокручивает страницу до блока `Идеи для поездок`.
2. Система показывает карточки направлений.
3. Гость изучает карточки `Восточные Гавайи`, `Домбай`, `Филиппины: сменить сезон`, `Ледяной Байкал`, `За сиянием — на Север`.
4. Гость использует кнопку `Посмотреть все`.
5. Система направляет пользователя в раздел `Путеводитель`.

**Альтернативы / исключения:**

- 2a. Блок не отображается.
  Пользователь не получает контентных подсказок по поездкам.
- 4a. Кнопка `Посмотреть все` отсутствует.
  Пользователь ограничен только карточками, показанными на главной странице.

### Прецедент: ExploreFaqKnowledgeBase

**ID:** UC-04

**Краткое описание:** гость сайта изучает популярные вопросы и использует FAQ как точку входа в базу знаний.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** База знаний `2read`.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- доступен блок `Вопросы и ответы`.

**Основной поток:**

1. Гость переходит к блоку `Вопросы и ответы`.
2. Система показывает карточки частых вопросов.
3. Гость изучает темы возврата и изменения билетов.
4. Гость использует кнопку `Посмотреть все`.
5. Система переводит пользователя в полный справочный раздел.

**Альтернативы / исключения:**

- 2a. Карточки FAQ не отображаются.
  Пользователь не получает быстрый доступ к популярным ответам.
- 4a. Кнопка `Посмотреть все` отсутствует или ведёт не в `2read`.
  Пользователь не может продолжить поиск ответа в полной базе знаний.

### Прецедент: ReviewServiceAdvantages

**ID:** UC-05

**Краткое описание:** гость сайта изучает преимущества сервиса в блоке `Фишки Туту`.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Маркетинговый контент главной страницы.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- блок `Фишки Туту` доступен пользователю.

**Основной поток:**

1. Гость прокручивает страницу до блока `Фишки Туту`.
2. Система показывает карточки преимуществ.
3. Гость изучает преимущества `Оплата позже`, `Кешбэк 3% баллами`, `Поддержка 24/7`, `Всё и сразу`.

**Альтернативы / исключения:**

- 2a. Одна или несколько карточек преимуществ отсутствуют.
  Пользователь не получает полное представление о ценности сервиса.

### Прецедент: EnterHotelDestination

**ID:** UC-06

**Краткое описание:** гость сайта вводит направление в форме поиска отелей.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Поисковая форма отелей.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- поле поиска отелей доступно для ввода.

**Основной поток:**

1. Гость активирует поле поиска отелей.
2. Система переводит фокус в поле ввода.
3. Гость вводит значение направления, например `Казань`.
4. Система сохраняет введённое значение в поле.

**Альтернативы / исключения:**

- 1a. Поле поиска недоступно для ввода.
  Пользователь не может начать поиск отелей.
- 4a. Введённое значение не сохраняется.
  Пользователь теряет введённые данные и не может продолжить сценарий.

### Прецедент: SubscribeToMailing

**ID:** UC-07

**Краткое описание:** гость сайта заполняет форму подписки и подтверждает согласие на обработку персональных данных.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Форма подписки.

**Предусловия:**

- открыта главная страница `Tutu.ru`;
- блок подписки отображается на странице.

**Основной поток:**

1. Гость переходит к блоку подписки.
2. Система показывает поле `Электронная почта`, кнопку `Подписаться` и чекбокс согласия.
3. Гость вводит email в поле подписки.
4. Система сохраняет введённое значение.
5. Гость активирует чекбокс согласия.
6. Система отмечает согласие как выбранное.

**Альтернативы / исключения:**

- 2a. Один из элементов формы отсутствует.
  Пользователь не может завершить сценарий подписки.
- 6a. Чекбокс не переключается.
  Пользователь не может подтвердить согласие и перейти к отправке формы.

### Прецедент: ReviewPrivacyPolicyFromSubscription

**ID:** UC-08

**Краткое описание:** гость сайта изучает юридическую информацию о персональных данных из формы подписки.

**Главные актёры:** Гость сайта.

**Второстепенные актёры:** Раздел юридической информации.

**Предусловия:**

- отображается форма подписки.

**Основной поток:**

1. Гость находит ссылку `Политика обработки персональных данных`.
2. Система предоставляет ссылку на юридический документ.
3. Гость использует ссылку для перехода к политике обработки данных.

**Альтернативы / исключения:**

- 1a. Ссылка отсутствует.
  Пользователь не может ознакомиться с юридическими условиями подписки.
- 2a. Ссылка не содержит корректного целевого адреса.
  Пользователь не попадает на нужный документ.

## Набор автоматизированных тестов

Архитектурно тесты построены через `Page Object`, но набор сценариев схлопнут до двух более насыщенных проверок:

- `TC-01. Навигация и контентные блоки главной страницы`
- `TC-02. Форма поиска отелей и лид-форма подписки`

### TC-01. Навигация и контентные блоки главной страницы

**Покрывает:** UC-01, UC-02, UC-03, UC-04, UC-05.

**Предусловие:** браузер запущен.

**Шаги:**

1. Открыть сохранённую главную страницу `Tutu.ru`.
2. Проверить состав и URL ссылок в шапке.
3. Проверить сервисный каталог и кнопку `Войти`.
4. Проверить блок `Идеи для поездок` и ссылку `Посмотреть все`.
5. Проверить блок `Вопросы и ответы` и ссылку `Посмотреть все`.
6. Проверить наличие карточек в блоке `Фишки Туту`.

**Ожидаемый результат:**

- пользователь видит целостную навигацию по сервису;
- контентные разделы главной страницы доступны;
- ссылки ведут в ожидаемые тематические разделы.

### TC-02. Форма поиска отелей и лид-форма подписки

**Покрывает:** UC-06, UC-07, UC-08.

**Предусловие:** открыта главная страница `Tutu.ru`.

**Шаги:**

1. Найти поле поиска отелей.
2. Ввести направление `Казань`.
3. Проверить, что значение сохранилось.
4. Перейти к блоку подписки.
5. Ввести email `invalid-email`.
6. Проверить доступность кнопки `Подписаться`.
7. Установить чекбокс согласия.
8. Проверить наличие ссылки на политику обработки персональных данных.

**Ожидаемый результат:**

- форма отелей принимает пользовательский ввод;
- блок подписки содержит все необходимые элементы;
- согласие переключается;
- юридическая ссылка доступна из формы подписки.

Команды запуска:

```bash
./gradlew uiChromeTest
./gradlew uiFirefoxTest
```

Базовый URL по умолчанию:

- `file:///Users/macbook/ITMO/TPO/docs/tasks/lab3/site/main.html`

Причина выбора snapshot-подхода:

- живой `tutu.ru` нестабилен в headless-режиме из-за внешних ресурсов, гидрации и антиботных механизмов;
- сохранённый HTML-снимок фиксирует реальный DOM главной страницы и позволяет воспроизводимо запускать Selenium-проверки;
- XPath-локаторы строятся по содержимому и атрибутам элементов, а не по их `id`.

Текущий статус в рамках данной среды:

- набор UI-тестов сокращён до 2 укрупнённых сценариев вместо нескольких тривиальных;
- сценарии покрывают 8 детализированных прецедентов;
- page object сохранён и расширен под навигацию, контентные блоки и форму подписки;
- `./gradlew testClasses` проходит успешно;
- `./gradlew uiChromeTest -Dheadless=true` проходит успешно;
- `./gradlew uiFirefoxTest -Dheadless=true` проходит успешно.

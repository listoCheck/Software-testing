# Отчёт по интеграционному тестированию

## Текст задания и система функций
Система функций:

```
x <= 0 :
(((((((sec(x) / cos(x)) * cos(x)) * csc(x)) ^ 3) ^ 2) / tan(x)) / ((((cot(x) ^ 3) * csc(x)) * ((cos(x) - cos(x)) * cot(x))) / tan(x)))

x > 0 :
(((((log_3(x) - log_3(x)) + (log_3(x) * log_10(x))) ^ 2) / log_10(x)) ^ 2)
```

Правила:
- Все тригонометрические функции выражены через базовую `sin(x)` (ряд Тейлора).
- Все логарифмические функции выражены через натуральный логарифм `ln(x)` (ряд).
- Упрощение исходной системы преобразованиями запрещено.
- Для каждого модуля есть табличные заглушки.

## UML-диаграмма
Файл диаграммы: `docs/uml.puml`.

## Области допустимых значений
- `ln(x)`, `log_b(x)`: `x > 0`.
- `sin(x)`, `cos(x)` определены для всех `x`.
- `tan(x)`: `cos(x) != 0`.
- `cot(x)`: `sin(x) != 0`.
- `sec(x)`: `cos(x) != 0`.
- `csc(x)`: `sin(x) != 0`.
- Тригонометрическая ветка системы: из-за `(cos(x) - cos(x))` знаменатель равен нулю для всех `x`, поэтому выражение не определено при `x <= 0`. Тесты фиксируют это как ожидаемое исключение.
- Логарифмическая ветка системы: `x > 0` и `log_10(x) != 0` (то есть `x != 1`).

## Стратегия интеграции
Стратегия — снизу вверх (Bottom-Up) с плавной заменой заглушек на реальные вычисления:
1. Листовые функции с моками зависимостей (`Cos`, `Tan`, `Cot`, `Sec`, `Csc`, `Logarithm`).
2. Подсистемы и маршрутизация с моками (`LogSystem`, `TrigSystem`, `SystemFunction`), чтобы зафиксировать сборку формул и выбор ветки без шума от рядов.
3. Частичная замена моков на реальные базовые модули: сначала подключаются `SeriesSin` и `SeriesLn`, затем на них собираются производные leaf-функции и только после этого подсистемы.
4. Поочерёдная замена веток в `SystemFunction`: отдельно подключается реальная log-ветка при моканой trig-ветке и отдельно реальная trig-ветка при моканой log-ветке.
5. Полная интеграция: обе ветки работают на реальных зависимостях, после чего выполняется проверка по CSV-эталонам.

Подробный план и критерии готовности: `docs/tasks/lab2/integration-strategy.md`.

## Описание тестового покрытия
Классы эквивалентности:
- Для системы: `x <= 0` и `x > 0`.
- Для логарифмов: `x <= 0` (ошибка), `x > 0` и `x != 1` (валидно), `x = 1` (деление на ноль в лог-ветке).
- Для тригонометрии: окрестности нулей `sin` и `cos` (ошибки в `cot`, `csc`, `sec`, `tan`).

Покрытие:
- `IntegrationLeafFunctionsWithMocksTest` — интеграция листовых модулей через моки зависимостей.
- `IntegrationSystemsWithMocksTest` — интеграция `LogSystem` и `TrigSystem` через моки.
- `IntegrationSystemRoutingWithMocksTest` — выбор ветки в `SystemFunction` через моки.
- `IntegrationTrigChainProgressiveTest` — постепенная сборка trig-цепочки: `SeriesSin -> Cos/Tan/Cot/Sec/Csc -> TrigSystem`.
- `IntegrationLogChainProgressiveTest` — постепенная сборка log-цепочки: `SeriesLn -> Logarithm -> LogSystem`.
- `IntegrationSystemProgressiveReplacementTest` — поочерёдное подключение реальных веток в `SystemFunction`.
- `IntegrationTrigFunctionsTest` — сравнение trig-цепочки на реальных значениях с `java.lang.Math`.
- `IntegrationLogSystemTest` — работа `LogSystem` на реальных значениях и граничных случаях.
- `IntegrationTrigSystemTest` — проверка корректной обработки деления на ноль в trig-ветке на реальных зависимостях.
- `IntegrationSystemRoutingTest` — маршрутизация полной системы на реальных зависимостях.
- `CsvAccuracyTest#systemMatchesReferenceForPositiveX` — прогон полной системы на реальных зависимостях по CSV-эталону.

## Графики по CSV-выгрузкам
Формирование CSV: `Main` + `CsvExporter`.

Пример команды:
```
./gradlew run --args="log 0.1 5 0.1 1e-9 docs/data/log.csv ;"
```

Скрипт для построения графиков: `docs/scripts/plot.py`.

Рекомендуемые графики:
- `ln(x)` на `(0.1..5)`.
- `logSystem(x)` на `(0.1..5)` с исключением `x=1`.
- Базовые тригонометрические функции (`sin`, `cos`) на `[-2π..2π]`.

## Выводы
1. Архитектура модулей позволяет не только Bottom-Up интеграцию, но и постепенную замену моков на реальные вычисления без резкого переключения всей системы.
2. Для каждой новой части сначала фиксируется контракт на моках, затем подставляется реальная базовая функция, затем собирается подсистема, и только потом она включается в `SystemFunction`.
3. Логарифмическая ветка корректно интегрируется через базовый `ln(x)` по ряду.
4. Тригонометрическая ветка системы не определена для `x <= 0` из-за `(cos(x) - cos(x))` в знаменателе; это отражено в тестах на каждом финальном этапе интеграции.
5. CSV-экспорт обеспечивает финальную проверку функций и построение графиков без модификации кода.

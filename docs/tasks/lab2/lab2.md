# Отчёт по интеграционному тестированию

## Текст задания и система функций
Система функций:

```
x <= 0 :
(((((((sec(x) / cos(x)) * cos(x)) * csc(x)) ^ 3) ^ 2) / tan(x)) / ((((cot(x) ^ 3) * csc(x)) * ((cos(x) - cos(x)) * cot(x))) / tan(x)))

x > 0 :
(((((log_3(x) - log_3(x)) + (log_3(x) * log_10(x))) ^ 2) / log_10(x)) ^ 2)
```

Правила:
- Все тригонометрические функции выражены через базовую `sin(x)` (ряд Тейлора).
- Все логарифмические функции выражены через натуральный логарифм `ln(x)` (ряд).
- Упрощение исходной системы преобразованиями запрещено.
- Для каждого модуля есть табличные заглушки.

## UML-диаграмма
Файл диаграммы: `docs/uml.puml`.

## Области допустимых значений
- `ln(x)`, `log_b(x)`: `x > 0`.
- `sin(x)`, `cos(x)` определены для всех `x`.
- `tan(x)`: `cos(x) != 0`.
- `cot(x)`: `sin(x) != 0`.
- `sec(x)`: `cos(x) != 0`.
- `csc(x)`: `sin(x) != 0`.
- Тригонометрическая ветка системы: из-за `(cos(x) - cos(x))` знаменатель равен нулю для всех `x`, поэтому выражение не определено при `x <= 0`. Тесты фиксируют это как ожидаемое исключение.
- Логарифмическая ветка системы: `x > 0` и `log_10(x) != 0` (то есть `x != 1`).

## Стратегия интеграции
Стратегия — снизу вверх с контролем через табличные заглушки:
1. Сборка `SystemFunction` с заглушками `TrigSystem` и `LogSystem` (проверка маршрутизации по `x`).
2. Интеграция `LogSystem` с заглушками `log3`, `log10`.
3. Замена заглушек на реальные `log3`, `log10`, построенные через `ln` (с `SeriesLn`).
4. Интеграция тригонометрических модулей через `SeriesSin` и производные функции.

## Описание тестового покрытия
Классы эквивалентности:
- Для системы: `x <= 0` и `x > 0`.
- Для логарифмов: `x <= 0` (ошибка), `x > 0` и `x != 1` (валидно), `x = 1` (деление на ноль в лог-ветке).
- Для тригонометрии: окрестности нулей `sin` и `cos` (ошибки в `cot`, `csc`, `sec`, `tan`).

Покрытие:
- `IntegrationSystemRoutingTest` — маршрутизация между ветками.
- `IntegrationLogSystemTest` — интеграция `LogSystem` с заглушками и проверка деления на `log10(x)`.
- `IntegrationTrigSystemTest` — проверка корректной обработки деления на ноль в триг-ветке.
- `AccuracyTest` — сравнение с `java.lang.Math` для `sin`, `ln`, `LogSystem`.

## Графики по CSV-выгрузкам
Формирование CSV: `Main` + `CsvExporter`.

Пример команды:
```
./gradlew run --args="log 0.1 5 0.1 1e-9 docs/data/log.csv ;"
```

Скрипт для построения графиков: `docs/scripts/plot.py`.

Рекомендуемые графики:
- `ln(x)` на `(0.1..5)`.
- `logSystem(x)` на `(0.1..5)` с исключением `x=1`.
- Базовые тригонометрические функции (`sin`, `cos`) на `[-2π..2π]`.

## Выводы
1. Архитектура модулей позволяет пошаговую интеграцию за счёт интерфейса `MathFunction` и табличных заглушек.
2. Логарифмическая ветка корректно интегрируется через базовый `ln(x)` по ряду.
3. Тригонометрическая ветка системы не определена для `x <= 0` из-за `(cos(x) - cos(x))` в знаменателе; это отражено в тестах.
4. CSV-экспорт обеспечивает проверку функций и построение графиков без модификации кода.

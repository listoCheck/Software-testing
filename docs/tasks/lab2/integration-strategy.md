# Стратегия интеграционного тестирования (Bottom-Up, Progressive Integration)

## Цель
Построить интеграционные тесты снизу вверх так, чтобы интеграция шла постепенно: новая логика сначала проверяется в изоляции на моках, затем реальные зависимости подменяют заглушки по одной, и только после стабилизации локальной цепочки модуль включается в следующую подсистему.

## Этапы
1. Листовые модули с моками зависимостей.
Проверяем, что модуль:
- вызывает нужные зависимости с корректными аргументами;
- корректно обрабатывает ветки с исключениями;
- стабилен до подключения реальных вычислений.

Покрытие:
- `Cos <- sin`
- `Tan <- sin, cos`
- `Cot <- sin, cos`
- `Sec <- cos`
- `Csc <- sin`
- `Logarithm <- ln`

2. Подсистемы с моками.
Проверяем сборку формулы и взаимодействие модулей без шума от рядов.

Покрытие:
- `LogSystem <- log3, log10`
- `TrigSystem <- sec, cos, csc, tan, cot`
- `SystemFunction <- trigSystem, logSystem`

3. Частичная замена моков на реальные базовые модули.
На этом шаге мы не включаем сразу всю реальную систему, а расширяем её по цепочке зависимостей:
- `SeriesSin` подставляется в `Cos`, затем в `Tan/Cot/Sec/Csc`, и только после этого собирается `TrigSystem`;
- `SeriesLn` подставляется в `Logarithm`, затем в `LogSystem`.

4. Поочерёдное включение реальных подсистем в `SystemFunction`.
Проверяем, что верхний уровень можно интегрировать плавно:
- сначала подключается реальная `LogSystem`, пока trig-ветка ещё остаётся моканой;
- затем подключается реальная `TrigSystem`, пока log-ветка ещё остаётся моканой;
- после этого обе реальные ветки собираются в `SystemFunction`.

5. Финальная верификация на реальных значениях.
После полной сборки системы прогоняем:
- сравнение leaf- и subsystem-цепочек с `java.lang.Math`;
- проверку граничных случаев (`x = 1` для log-ветки, `x <= 0` для trig-ветки);
- CSV-эталоны для полной positive-ветки системы.

## Критерии готовности
- Сначала зелёные тесты на моках: контракты модулей и маршрутизация подтверждены.
- Затем зелёные progressive-тесты: реальные `SeriesSin` и `SeriesLn` успешно входят в leaf-цепочки и подсистемы без резкого переключения всей системы.
- После этого зелёные полные интеграционные тесты на реальных значениях.
- Ветка `x <= 0` стабильно даёт ожидаемую ошибку для текущей формулы trig-части.
- Ветка `x > 0` совпадает с эталонными значениями в допусках и по CSV-референсам.
